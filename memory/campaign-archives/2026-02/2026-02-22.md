### Claude Sonnet 4.6 挂死问题 — 根因定位与修复
- **现象**：切换到 Claude 后无响应，系统"挂死"。
- **根因 1 (参数冲突)**：`temperature` 设为 1 时必须配合推理模式（Extended Thinking），否则 Anthropic API 直接返回 400 错误，Gateway 死循环重试。
- **根因 2 (超时太短)**：Claude 响应第一个 token 就需要 15-20s，之前设的 25s 超时导致每次都被截断。
- **修复**：`temperature` 锁定为 1（配合推理模式），`timeout` 从 25s 延长至 120s。
- **教训**：Claude 的 `temperature=1` 必须与 `thinkingDefault` 推理设置配套，缺一不可。

## 每日学习总结 (2026-02-22)
- **参数陷阱**：Claude 推理模式强制要求 temperature=1，且响应时间长，必须延长 timeout 至 120s 以上，否则会造成系统挂死假象。
- **后勤优先级**：在无人值守系统中，系统级内存清理（Memory Purger）比应用级监控（Watchdog）更重要，能有效防止连环重启。
- **工具链闭环**：通过 Colima 成功在 M 系列 Mac 上跑起 Docker 容器，补齐了本地测试短板。
- **搞钱节奏**：面对高标准 AI 评审，应采用“原子化/防御性编程”快速迭代，而不是死磕复杂的认证流程。
- **并发管理**：多 Agent 架构（Investor vs Main）能有效隔离任务上下文，提升响应质量。
